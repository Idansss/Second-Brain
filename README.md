# Second Brain

AI memory for your life and work: capture fast, extract meaning automatically, and recall it when it matters.

## What this repo contains

- **Web app**: capture notes, search, tasks, chat, profile/settings
- **Database**: Postgres (Supabase) + Drizzle schema + migrations + pgvector embeddings
- **AI**:
  - Claude (`claude-sonnet-4-6`) for extraction + chat + digest
  - Voyage AI (`voyage-3`, 1024 dims) for embeddings
  - Jina Reader for URL ‚Üí markdown scraping (no key needed)

## Getting started (local dev)

### Prereqs

- Node.js **18+**
- A [Supabase](https://supabase.com) project
- [Anthropic API key](https://console.anthropic.com) (Claude)
- [Voyage AI API key](https://dash.voyageai.com) (embeddings)

### Install

```bash
npm install
```

### Environment variables

Copy the template to the web app:

```bash
cp .env.example apps/web/.env.local
```

Required values (web app):

```env
# Supabase ‚Üí Settings ‚Üí API
NEXT_PUBLIC_SUPABASE_URL="https://PROJECT.supabase.co"
NEXT_PUBLIC_SUPABASE_ANON_KEY="sb_publishable_..."

# Supabase ‚Üí Settings ‚Üí Database ‚Üí Connection string ‚Üí Transaction mode
# IMPORTANT: must include ?sslmode=require
DATABASE_URL="postgresql://postgres.<ref>:<password>@aws-0-<region>.pooler.supabase.com:6543/postgres?sslmode=require"

# AI
ANTHROPIC_API_KEY="sk-ant-..."
VOYAGE_API_KEY="pa-..."
```

### Supabase setup

#### 1) Enable pgvector

Run in **Supabase ‚Üí SQL Editor**:

```sql
create extension if not exists vector;
```

#### 2) Run DB migrations

You have two options:

- Option A (CLI):

```bash
cd packages/db
npm run db:generate
npm run db:migrate
```

- Option B (SQL Editor):
  - Open and run the SQL file in `packages/db/migrations/` (generated by Drizzle).

#### 3) Storage bucket for avatars (profile pictures)

Create a bucket named **`avatars`** in **Supabase ‚Üí Storage**.

Then either:

- **Make it public** (simplest), or
- Keep it private and add RLS policies (recommended)

RLS policies (run in **SQL Editor**):

```sql
-- Allow authenticated users to upload/update their avatar objects
create policy "Allow authenticated avatar uploads"
on storage.objects
for insert
to authenticated
with check (bucket_id = 'avatars');

create policy "Allow authenticated avatar updates"
on storage.objects
for update
to authenticated
using (bucket_id = 'avatars');

-- Allow authenticated users to read avatars (required for images to display)
create policy "Allow authenticated avatar reads"
on storage.objects
for select
to authenticated
using (bucket_id = 'avatars');
```

### Start dev server

From the repo root:

```bash
npm run dev
```

Web app: `http://localhost:3000`

## How it works

### Capture ‚Üí enrich (async)

When you create a note:

1. The note is saved immediately (fast capture)
2. A background job:
   - Scrapes URLs via Jina Reader (for link notes)
   - Extracts intelligence via Claude (summary, key points, tags, entities, tasks)
   - Generates embeddings via Voyage AI (`voyage-3`, 1024 dims)
3. Derived data is stored back into Postgres and linked via join tables

### Search / chat (RAG)

1. Embed the query (Voyage)
2. Use pgvector cosine similarity to retrieve top notes
3. Send retrieved notes + question to Claude
4. Stream back an answer with `[NOTE N]` citations

## Troubleshooting

### tRPC requests return 500 + `getaddrinfo ENOTFOUND db.<ref>.supabase.co`

Your `DATABASE_URL` is wrong. Use the **Transaction mode / pooler** connection string from:
**Supabase ‚Üí Settings ‚Üí Database ‚Üí Connection string ‚Üí Transaction mode**.

### tRPC requests return 500 and env validation complains about `sslmode`

Your `DATABASE_URL` must include **`?sslmode=require`**.

### `password authentication failed for user "postgres"` or ‚ÄúCircuit breaker open‚Äù

- Your DB password in `DATABASE_URL` is wrong (or includes extra characters like `[` `]`)
- Reset the DB password in Supabase and paste the exact value into `DATABASE_URL`
- Restart the dev server (the circuit breaker clears on process restart)

### Avatar uploads succeed but profile picture never displays

Add the **Storage SELECT policy** for the `avatars` bucket (see ‚ÄúStorage bucket for avatars‚Äù above), or make the bucket public.

### `EADDRINUSE: address already in use :::3000`

Something is already listening on port 3000. Stop it, or run Next on a different port.
